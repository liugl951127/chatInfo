模块依赖图Module Dependency Graph · Maven Multi-Module

cs-common 是整个架构的「工具底座」,所有服务都依赖它提供的 JWT 拦截器、Redis 配置、UserContext ThreadLocal、ApiResponse 包装器、跨域配置。cs-gateway 不依赖业务服务,只做路由与全局鉴权过滤;业务服务之间 解耦(通过 Redis Pub/Sub 和 STOMP 间接通信,不直接 RPC 调用)。

↑ 实线箭头 = Maven compile 依赖 业务服务不直接 RPC, 通过 Redis Pub/Sub + STOMP 间接联动

i模块职责矩阵

基础层

cs-common

JWT 签发/校验、Redis 客户端配置、UserContext ThreadLocal、跨域配置、ApiResponse 包装、Knife4j 配置。

基础层

cs-gateway

Spring Cloud Gateway,AuthGlobalFilter 在网关层统一校验 JWT,把 X-User-Id / X-User-Role / X-User-Nickname 头透传给下游。

基础层

cs-auth

注册 / 登录 / /auth/me。HS512 签发 24h JWT。BCrypt 校验密码。下游服务通过 JwtAuthInterceptor 二次校验。

基础层

cs-im

STOMP WebSocket 端点、会话生命周期、消息持久化、抢单 (CAS)、CSAT 评分、录像 (record-sdk 服务端)。

智能层

cs-cdp

客户主档案 + 行为事件流 + 标签计算 (30+ 标签,阶段 1)。提供 /api/cdp/profile/{uid} 给前端 360 视图。

智能层

cs-prediction

订阅 cs-cdp 事件流,RuleEngine + M3 推理,触发主动推送 (PUSH / SESSION_INVITE / EMAIL),落库 prediction_event。

智能层

cs-customer-success

客户健康分 0-100 实时计算(阶段 3),Onboarding 漏斗,续约预测,坐席工作量统计。

智能层

cs-community

客户社区 UGC (发帖/回复/点赞),AI 主持人自动审核归类,积分/勋章激励,优质 UGC 反哺 RAG 知识库。

渠道层

cs-video

WebRTC 信令 (基于 STOMP),P2P 1v1 通话 + 屏幕共享,阶段 3 引入 mediasoup SFU 支撑多方。复用 cs-im 的 RecordService。

数据流图Data Flow · End-to-End Request Lifecycle

一次典型请求的完整生命周期:用户浏览器 → Vue SPA (axios / STOMP) → cs-gateway (JWT 校验 + 头注入) → 业务微服务 (Service + Mapper) → MariaDB 持久化 + Redis 缓存/广播 → 通过 Pub/Sub 反向推回前端。展现了 JWT 透传、STOMP 双向通信、Redis Pub/Sub 跨实例推送、离线消息兜底 四个关键链路。

关键链路 ① 客户发消息 → ② cs-im 处理 → ③ Redis Pub/Sub 广播 → ④ STOMP 推回坐席;同时 cs-prediction 监听 cdp 事件,主动通过 STOMP 推送给客户(无需客户发起)。
05

部署架构Deployment · Docker Compose + Kubernetes

开发期用 Docker Compose 单机编排(12 容器),生产期迁移到 Kubernetes 集群(8 节点)。两种形态共用同一套镜像,只调整 application.yml 的 Nacos/Redis/MariaDB 地址。所有服务 无状态化(除 m3-adapter 的 Python 进程),支持水平扩展。

Docker Compose (开发)
Kubernetes (生产)
./records:/data/records (cs-im) · ./conf:/conf (mariadb init) · ./m3-py:/app (m3-adapter) Kubernetes (Production · 8 nodes) Namespace: chat-prod · 镜像复用 compose · StatefulSet 只用于 mariadb / redis / nacos CHAT-PROD NAMESPACE Deployment cs-gateway / cs-auth replicas: 2 · HPA Deployment cs-im · cs-cdp replicas: 3 · HPA Deployment cs-pred / success replicas: 2 · HPA Deployment cs-video / voice replicas: 2 Deployment m3-adapter replicas: 1 (CPU 限) StatefulSet mariadb · 3 副本 PVC 100Gi StatefulSet redis-sentinel 1 master + 2 replica StatefulSet nacos · 1 副本 + MySQL 后端 DaemonSet coturn (TURN) 每节点一个 Service / Ingress nginx-ingress chat.example.com
关键决策:无状态服务(cs-im / cs-cdp 等)用 Deployment + HPA 水平扩;有状态存储(MariaDB / Redis / Nacos)用 StatefulSet + PVC 持久化;coturn用 DaemonSet 每节点一份以最小化 STUN 延迟。
